WKWebView - Local file

Update: In iOS 10 (Not sure when this started working) Local files and assets load when baseUrl is set.

Currently in iOS8.1/8.2 beta there is a bug with WKWebView that blocks the loading of local files.

One work around is to embed a minimal web server (in this case OCFWebServer) in the app and serve files locally ie: http://localhost:8080

This html file is located in the bundle, and loads a local css style sheet from the assets sub directory.

A ScriptMessageHandler "MyObserver" is added when the WKWebView is configured at start up. The "Call Javascript" button calls the javascript function testJS() in this html file.

window.webkit.messageHandlers.MyObserver.postMessage(message) calls the Objective-C method: userContentController:(WKUserContentController *)userContentController didReceiveScriptMessage:(WKScriptMessage *)message and brings up a dialog with the passed message text.
